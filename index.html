Make changes and submit a Pull Request

You can edit files using the Github GUI, make changes, and then submit a Pull Request to Merge your changes into the main branch. Others can review your changes and make suggestions for improvements or ask questions before you merge - while in the meantime - they can have their own separate branches with changes they are working on. This allows a bunch of people to work on the same project together without issues. By the way git doing the actual "work" of allowing you to make branches, track and merge changes, Github is really just a place to put your project (your "git repository").
